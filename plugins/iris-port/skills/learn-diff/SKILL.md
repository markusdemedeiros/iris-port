---
name: learn-diff
description: Instruct the lean-style-critic to study a reviewed git diff and adapt its memory to match the house style. Use after a human has manually cleaned up ported Lean files and you want the critic to internalize the patterns.
---

Instruct the lean-style-critic to read a git diff generated by manual review. The purpose is to allow the lean-style-critic to learn the house style. Diffs are stored in the project's `diffs/` directory and the user will tell you which to read. Below is a prompt you can use to call the subagent:

    The user has provided a git diff at ${CLAUDE_PROJECT_DIR}/diffs/[DIFF.txt] showing their manual style fixes to iris-lean files. Read the diff yourself for full details. Below is a high-level summary of the patterns in their changes. Your job is to generalize these into house style rules (not specific to the files they were working on) and save them to your memory so you apply them in all future reviews.

    Summary of changes in the diff

    [ SUMMARY GOES HERE ]

    Your task

    1. Read the diff file at ${CLAUDE_PROJECT_DIR}/diffs/[DIFF.txt] to see the full details.
    2. Read your existing memory files (and the seed rules at ${CLAUDE_PLUGIN_ROOT}/notes/HOUSE_STYLE.md) to avoid duplicating what you already know.
    3. Generalize all the above patterns into house style rules that apply to any Lean 4 code in this project.
    4. Save these rules to your local agent memory so they are loaded in every future review.
    5. Report back with the list of generalized rules you saved.
